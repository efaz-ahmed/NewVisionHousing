# New Vision Housing — Website V1

Single-page marketing + investor-relations site for **New Vision Housing Ltd (NVH)**, a UK private limited company offering ethical, share-based, debt-free property investment grounded in Islamic finance principles.

## Live file

`index.html` — self-contained HTML (~10 MB) with all images, fonts, and assets inlined via a custom bundler. No external dependencies at runtime.

## Sections

1. **Hero** — brand promise, investor CTA, stat counters (shareholders / AUM / properties)
2. **Belief** — ethical framework grounded in Shariah-compliant investing
3. **Journey** — pinned horizontal-scroll timeline of NVH milestones from 2017 onward
4. **Portfolio** — core holdings (residential + commercial UK property) + sister-company ventures
5. **Vision 10** — upcoming flagship venture (Matrimony Ltd) and roadmap
6. **Benefits** — shareholder benefits including Hajj Pot, Wedding Pot, AGM participation
7. **Invest** — pathway, partnership structure, and contact form (`#get-in-touch`)

## Build pipeline

The site is generated by a chain of Python build scripts that progressively patch the source HTML. The current head is `build_v34.py` which consolidates seven layered patch scripts into a single mobile-only style block plus a MutationObserver-based map renderer.

```bash
python build_v34.py    # reads IndexV33.html, writes index.html
```

## Mobile responsiveness

Breakpoints used:
- `≤1080px` — drawer takes over from top-nav
- `≤900px` — portfolio core-holdings card switches to vertical stack
- `≤768px` — hero BG fix, flagship image hidden
- `≤640px` — hero typography centred + bold + 35% larger; stats become single row

## Contact form

The Get-in-touch form posts to `/api/contact` (Vercel serverless
function in [`api/contact.js`](./api/contact.js)). The function
validates the submission and sends an email via [Resend](https://resend.com).

### Deployment (Vercel)

1. **Connect this repo to Vercel** (vercel.com → New Project → import GitHub repo).
2. In the Vercel project, go to **Settings → Environment Variables** and add:

   | Name | Value | Required |
   |---|---|---|
   | `RESEND_API_KEY` | Your Resend API key (`re_…`) — generate at https://resend.com/api-keys | **yes** |
   | `CONTACT_RECIPIENT` | Email address that receives submissions. Default: `efaz.mintu@gmail.com` (testing). Set to `info@nvhltd.com` for production. | no |
   | `RESEND_FROM` | The "From:" header Resend sends. Default: `NVH Website <onboarding@resend.dev>` (Resend's shared sender — works without domain verification, lower deliverability). For production, verify `nvhltd.com` in Resend's dashboard and set this to e.g. `NVH <contact@nvhltd.com>`. | no |

3. Vercel auto-deploys on every push to `main`. No build step needed (static HTML + zero-dep serverless function).

### Security

The API key never lives in the repo — it's only in Vercel's encrypted env vars. The `.gitignore` also excludes any local `.env`/`.env.local` files in case you ever run `vercel dev` locally.
